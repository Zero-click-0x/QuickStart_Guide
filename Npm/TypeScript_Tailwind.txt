=> input.css contains your Tailwind directives and custom CSS and Tailwind compiler read that css class.
=> output.css comes with (CSS reset + normalization) is the compiled/processed file generated by Tailwind CLI, and index.html links to output.css to use the styles.

1. You write classes in HTML → <div class="bg-blue-500">
2. Tailwind CLI scans HTML files → finds "bg-blue-500"
3. Reads input.css directives → @tailwind utilities
4. Generates into output.css → .bg-blue-500 { background-color: ... }
5. Browser loads output.css → styles applied
